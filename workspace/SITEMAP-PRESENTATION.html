3. 회고 시스템 /retrospective

개인 회고 작성 /new

입력 필드

  • 프로젝트명 (Select / 직접 입력)
  • 회고 기간 (DateRangePicker)
  • 작성자 이름 (자동 채움)
  • 역할 (복수 Checkbox): 디자인, 인쇄, 웹사이트 제작, 브랜딩, 촬영, 중계, 라이브 중계, 사진촬영, 진행, 홍보, PR
  • KPT: Keep / Problem / Try (Textarea, 항목 추가)
  • SSC: Start / Stop / Continue (Textarea, 항목 추가)
  • 공유 메모 (선택)

동작

  • 임시 저장 (Draft) 지원
  • 제출 후 수정 불가 (super/boss만 허용)

취합 뷰 /summary/[projectId]

기능

  • 프로젝트 선택 → 회고 전체 조회
  • 역할별 참여 현황 테이블
  • 통합 인사이트 자동 묶기
  • 역할별 핵심 요약 테이블
  • 최종 합의 사항 편집 (manager+)

통합 인사이트 그룹핑

Keep + Continue = 강점 유지 Problem + Stop = 개선 필요 Try + Start = 다음 액션
shadcn/ui
TableTabsCardBadgeTextarea

7. 클라이언트 반응 예측 /predict Gemini API

실제 대화 데이터 기반으로 AI 아바타를 생성하고, 예상 반응을 시뮬레이션

아바타 목록 /avatars

  • 아바타 카드 그리드
  • 이름, 소속, 학습 상태 표시
  • 1인당 여러 아바타 생성 가능
학습 상태
미학습 학습중 학습완료 업데이트 필요

아바타 생성 /new

Step 1: 기본 정보

  • 이름 (예: 김과장)
  • 소속/회사명, 직급/역할
  • 아바타 이미지

Step 2: 대화 데이터 학습

  • 카톡/메신저 내보내기 업로드
  • 이메일 스레드 붙여넣기
  • 회의 메모/통화 기록 직접 입력
  • 기존 회의록에서 발언 자동 추출
파싱 미리보기 → 발화자별 분리 확인 → 학습 실행

인격 프로필 /settings

AI 자동 추출 + 수동 보정

  • 말투: 격식/비격식, 직설/완곡, 간결/장문
  • 성격 키워드 (꼼꼼, 가격민감 등)
  • 관심사/자주 언급 주제
  • 의사결정 패턴 (즉답/검토/상사확인)
  • 민감 주제 (예산, 일정 지연)

Step 4: 테스트 대화

  • 설정 완료 후 바로 테스트
  • 불만족 시 Step 2~3 반복
