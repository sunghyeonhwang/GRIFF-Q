# GRIFF-Q 0.1 사내 업무 관리 프로그램 기획 정리

## 1) 목적
현재 Asana / Slack / Google Sheets로 분산된 업무를 하나의 내부 시스템으로 통합한다.

## 2) 공통 요구사항
- 멀티 계정 기반 동시 접속 및 페이지(기능)별 공동 수정 지원
- 백업 체계
  - 파일 백업
  - 루틴(정기) 백업
  - 클라우드 백업
- 변경 이력 관리
  - 작업자별 수정 로그 기록
  - 관리자 권한에 따른 로그 조회
  - 롤백 기능 → v0.2 이관 (v0.1은 조회만)
- 관리자 권한 타입 4종
  - `super`
  - `boss`
  - `manager`
  - `normal`

## 3) 핵심 기능 페이지
### A. 로그인 / 로그아웃
- 계정 인증 및 세션 관리

### B. 대시보드
- 전체 업무 현황 요약
- 주요 마감일, 요청 건수, 최근 변경 로그 표시

### C. 페이지 1: 회의록 관리
- 기존 정리 기준: Google Sheet
  - https://docs.google.com/spreadsheets/d/11_CTo7otgQbVu5sFkZVQRMQpE5vp2E00AREeexhyhr8/edit?usp=sharing
- 개선 목표
  - 더 직관적인 표 형태
  - 액션 아이템 입력 폼
  - 마감일 입력 및 관리

### D. 페이지 2: 입금/결제 요청
- 입력 필드
  - 이름
  - 금액
  - 은행
  - 계좌번호
  - 입금자명
  - 마감일
  - 비고
- 출력
  - 입금할 은행명 중심의 리스트 페이지

### E. 페이지 3: 견적서 공동 작업
- 기능
  - 프로젝트/항목/초안/비고 등 상세 작성
  - 순차 편집 + 잠금 방식 공동 작업 (편집 시 다른 사용자에게 잠금 표시)
  - 퍼블리시: PDF / Google Sheets / Excel
  - 작년 견적 비교
  - 하이라이트 기능
- v0.2 예정: Supabase Realtime+Presence 기반 실시간 동시 편집

### F. 클라이언트 반응 예측 시스템 (v0.1 MVP)
- 클라이언트별 수동 프리셋 페르소나 설정 (이름, 소속, 직급, 성격 키워드, 말투 스타일)
- Gemini API 연동 채팅 UI
- v0.2 예정: 대화 데이터 업로드 → 자동 인격 프로필 생성, 감정 분석 패널, 상황 시뮬레이션

### G. 회고 시스템
- 프로젝트 팀 회고를 위한
  - 개인 작성 양식
  - 최종 취합/정리 페이지

## 4) 회고 시스템 상세 요구사항

### 4-1. 적용 프레임워크
- KPT + SSC 혼합
  - KPT: Keep / Problem / Try
  - SSC: Start / Stop / Continue

### 4-2. 개인 회고 양식 템플릿 (1종)
- 작성자 이름 포함
- 역할(Role) 포함
- 역할은 복수 선택 가능
  - 디자인
  - 인쇄
  - 웹사이트 제작
  - 브랜딩
  - 촬영
  - 중계
  - 라이브 중계
  - 사진촬영
  - 진행
  - 홍보
  - PR

### 4-3. 취합용 최종 정리 템플릿 (1종)
- 7명 팀원 작성본 취합
- 역할별/주제별로 묶어 회의 공유 가능해야 함

### 4-4. 폼 요구사항
- 입력 폼 필요
- 뷰(열람) 폼 필요

## 5) 제약사항 (Constraints)
- 팀원 수: 본인 포함 7명
- 정량 평가 없음
- 정성 평가만 수행
- 저장 위치: `workspace/Retrospective/`

## 6) 비목표 (Non-goals)
- 실시간 입력 폼
- 프로젝트 관리 도구 직접 연동
- v0.1 범위 외 (v0.2 이관 항목)
  - 견적서 실시간 동시 편집 (Supabase Realtime)
  - 아바타 대화 데이터 자동 학습 시스템
  - 감사 로그 롤백 기능
  - 백업 자동화

## 7) 문서 스타일 가이드
- 톤: 실무적이고 간결
- 분량: 작성자가 부담 없이 채울 수 있는 수준
