| 태스크 보류 | 없음 | **on_hold 상태 추가** (프로젝트 연동 + 개별 보류) |

| 글로벌 검색 | 없음 | **Cmd+K 통합 검색** |
| 지원 메뉴 범위 | 미정의 | **회고만 v0.3 연동**, 나머지는 미포함 |
| AI 리뷰 | 후순위 | **v0.3 포함** — 버튼 1개로 AI 리뷰 자동화 |
| 알림 채널 | 미정의 | v0.3 인앱 알림, **v0.4에서 슬랙 연동** |
| action_items | 회의록과 별도 | **액션아이템 = 태스크** 자동 연결 |

## 2) 시스템 권한 4단계

| 권한 | 대시보드 | 스크럼 | 태스크 | 프로젝트 | 일정 |
|---|---|---|---|---|---|
| **대표** | 전사 현황 열람 | 본인 작성 + **전체 열람** | **전체 열람**(읽기) + 본인 관리 | 생성/열람/승인 | 전사 열람+생성+**전체 수정** |
| **관리자** | 전사 현황 열람 | 본인 작성 + **전체 열람** | **전체 열람**(읽기) + 본인 관리 | 생성/열람/승인 | 전사 열람+생성+**전체 수정** |
| **PM** | 담당 프로젝트 현황 | 본인 작성 + **팀원 열람** | 본인+**담당 프로젝트 팀원** 관리 | 생성/열람/승인(담당) | 전사 열람+생성 |
| **일반** | 본인 현황 | 본인 작성만 | 본인 관리+직접 생성 | 생성/열람(배정된 것) | 전사 열람+생성 |

### 승인 권한 상세

| 프로젝트 유형 | 완료 승인 가능자 |
|---|---|
| **미니프로젝트** | PM 단독 또는 관리자 |
| **프로젝트** | PM + 관리자 합의 (둘 다 승인 필요) |
| **셀프 프로젝트** | 생성자 본인 확인 (승인 불필요, 결과보고로 대체) |

## 3) 프로젝트 유형 3종 — 업무 흐름

### 유형 비교표

| 항목 | 미니프로젝트 | 프로젝트 | 셀프 프로젝트 |
|---|---|---|---|
| **트리거** | 클라이언트 요청 | 클라이언트 요청 | 내부 자체 기획 |
| **기획 단계** | 간단 기획 | 상세 기획 + 요구사항 확인 | 없음 (바로 태스크) |
| **킥오프** | ❌ 없음 | ✅ 필수 | ❌ 없음 |
| **태스크 생성** | 기획자 or 유저 직접 | 기획자 or 유저 직접 | 기획자 or 유저 직접 |
| **완료 승인** | PM/관리자 | PM + 관리자 합의 | 본인 확인 |
| **포스트모템** | 선택 | 필수 | ❌ 없음 |
| **결과보고** | — | — | ✅ 팀 전체 공유 |
| **예상 기간** | 1~5일 | 1주~수개월 | 자유 |
| **팀 규모** | 1~3명 | 3~10명 | 1~2명 |

### 3-1. 미니프로젝트 흐름

```
클라이언트 요청
    │
    ▼
[기획] 간단 기획 (요구사항 정리)
    │
    ▼
[태스크 생성] 기획자가 태스크 배정 / 유저가 직접 생성
    │
    ▼
[업무 수행] 담당자가 태스크 진행
    │
    ├─ 수정 필요 → 태스크 상태: review → 수정 → 재진행
    │
    ▼
[완료 판단] PM 또는 관리자가 승인 버튼 클릭
    │
    ▼
[완료] 프로젝트 상태: completed
    │
    └─ [포스트모템] 선택사항 — 짧은 회고 가능
```

### 3-2. 프로젝트 흐름

```
클라이언트 요청
    │
    ▼
[기획] 상세 기획서 작성
    │
    ▼
[요구사항 확인] 클라이언트와 요구사항 정리/합의
    │
    ▼
[기획 업데이트] 확인된 요구사항 반영
    │
    ▼
[킥오프] ★ 필수
    │  · 목표/범위/제약조건/성공기준 설정
    │  · R&R 배정 (PM/디자인/영상/일반 등)
    │  · 마일스톤 설정
    │  · 체크리스트 확인
    │
    ▼
[태스크 생성] 기획자가 태스크 배정 / 유저가 직접 생성
    │
    ▼
[업무 수행] 담당자가 태스크 진행
    │
    ├─ 수정 필요 → 태스크 상태: review → 수정 → 재진행
    │
    ▼
[완료 판단] PM 승인 + 관리자 승인 (둘 다 필요)
    │
    ▼
[완료] 프로젝트 상태: completed
    │
    └─ [포스트모템] ★ 필수 — 기존 회고 모듈 연동
```

### 3-3. 셀프 프로젝트 흐름

```
[태스크 생성] 기획자 또는 유저가 직접 태스크 생성
    │  · 킥오프 없음, 간단한 목표만 설정
    │
    ▼
[업무 수행] 담당자가 태스크 진행
    │
    ├─ 수정 필요 → 수정 → 재진행
    │
    ▼
[확인] 본인이 완료 확인 (승인 버튼 불필요)
    │
    ▼
[완료] 프로젝트 상태: completed
    │
    └─ [결과보고] ★ 필수 — 팀 전체 공유
       · 간단 요약 (뭘 했는지, 결과물, 배운 점)
       · 팀 피드 또는 공유 페이지에 게시
```

## 4) 핵심 메뉴별 시나리오

> 각 메뉴에 대해 역할별 주요 사용 시나리오를 정의합니다.

---

### 4-1. 대시보드 (/dashboard)

#### 메뉴 정의
> 로그인 후 첫 화면. 역할에 따라 보이는 정보 범위가 다름.

#### 시나리오: 대표

```
[대표] 출근 → 대시보드 확인
    │
    ├─ 전사 프로젝트 현황 (진행중/완료/지연 건수)
    ├─ 이번 주 주요 마일스톤
    ├─ 팀원 스크럼 완료 현황 (누가 했고 누가 안 했는지)
    └─ 오늘 전사 일정 요약

    → "지연 프로젝트가 2건이네" → 클릭 → 프로젝트 상세로 이동
```

#### 시나리오: 관리자

```
[관리자] 출근 → 대시보드 확인
    │
    ├─ 담당 프로젝트 그룹 진행률 (전사 범위)
    ├─ 승인 대기 항목 (완료 승인 요청 n건)
    ├─ 팀원 부하 현황 (누가 태스크가 많은지)
    ├─ 이번 주 마감 태스크
    └─ 오늘 일정 요약

    → "승인 대기 2건" → 클릭 → 승인 처리
```

#### 시나리오: PM

```
[PM] 출근 → 대시보드 확인
    │
    ├─ 내 프로젝트 진행률 (담당 프로젝트만)
    ├─ 오늘의 태스크 (내가 배정한 것 + 내 것)
    ├─ 팀원 스크럼 현황 (내 프로젝트 팀원만)
    ├─ 승인 대기 항목 (내 프로젝트 완료 요청)
    └─ 오늘 일정

    → "팀원 A가 스크럼 미완료" → 리마인드 필요 인지
```

#### 시나리오: 일반 (디자인/영상/팀원)

```
[일반] 출근 → 대시보드 확인
    │
    ├─ 오늘의 태스크 (나한테 배정된 것)
    ├─ 내 스크럼 상태 (오늘 작성했는지)
    ├─ 마감 임박 태스크
    └─ 오늘 일정

    → "스크럼 미작성" → 바로 스크럼 시작 버튼
    → "마감 임박 태스크 2건" → 클릭 → 태스크 상세
```

### 4-2. 데일리 스크럼 (/scrum)

#### 메뉴 정의
> 매일 아침 "오늘 하루를 설계"하는 3단계 플로우. 전 직원 대상.

#### 시나리오: 일반 사용자 (전 역할 공통)

```
[사용자] 출근 → 데일리 스크럼 시작
    │
    ▼
[A단계: 브레인스토밍] "오늘 뭘 해야 하지?"
    │  · AI가 질문: "오늘 뭘 할 예정이에요?"
    │  · 자유 입력으로 할 일 나열
    │  · 어제 미완료 태스크 자동 이월 제안: "어제 이것도 남았는데?"
    │  · 마감 임박 프로젝트 태스크 자동 표시
    │  · 오늘 예정 회의 자동 표시
    │  → 결과: Raw List (정리 안 된 할 일 목록)
    │
    ▼
[B단계: 우선순위 정하기] "뭐가 더 급하지?"
    │  · AI가 마감일/중요도 기반 우선순위 추천
    │  · 드래그로 순서 변경
    │  · 긴급(빨강) / 중요(주황) / 일반(회색) / 후순위(파랑) 라벨링
    │  → 결과: Prioritized List
    │
    ▼
[C단계: 시간 배치] "언제 하지?"
    │  · 09:00~18:00 타임라인에 드래그 배치
    │  · 예상 소요시간 선택 (15분/30분/1시간/2시간/반나절)
    │  · 오늘 회의 일정 자동 블록 (회색 블록)
    │  · 과부하 경고: "할당 8시간인데 10시간 넣으셨어요"
    │  → 결과: Today's Schedule
    │
    ▼
[완료]
    · 스크럼 항목 → 태스크에 자동 등록/연결
    · 대시보드 "오늘의 태스크"에 반영
```

#### 시나리오: PM이 팀원 스크럼 확인

```
[PM] 데일리 스크럼 페이지 → 팀 현황 탭
    │
    ├─ 팀원별 스크럼 완료/미완료 상태
    ├─ 각 팀원의 오늘 계획 요약 (클릭하면 상세)
    └─ 미완료자에게 리마인드 (알림 전송)
```

#### 시나리오: 스크럼 건너뛰기 (휴가/외근/출장)

```
[사용자] 데일리 스크럼 페이지 → "오늘 건너뛰기"
    │
    ├─ 사유 선택: 휴가 / 외근 / 출장 / 병가 / 기타
    │
    ▼
[결과]
    · 스크럼 상태: skipped (사유 포함)
    · 대시보드에서 "스크럼 건너뜀 (휴가)" 표시
    · PM 팀 현황에서 사유와 함께 표시 (미완료와 구분)
    · 어제 미완료 태스크는 내일로 자동 이월
```

#### 시나리오: 관리자/대표가 팀원 스크럼 열람

```
[관리자/대표] 데일리 스크럼 → 팀 현황 탭
    │
    ├─ 전체 팀원 스크럼 상태 (완료/미완료/건너뜀)
    ├─ 팀원 클릭 → 해당 팀원의 오늘 스크럼 상세 열람
    │   (읽기 전용 — 수정 불가)
    └─ 팀 전체 완료율 통계
```

#### 시나리오: 하루 마감 회고 (퇴근 시)

```
[사용자] 퇴근 전 → 스크럼 마감 회고
    │
    ├─ 오늘 계획한 항목 중 완료/미완료 체크
    ├─ 미완료 사유 간단 입력 (회의 길어짐, 긴급 요청 등)
    └─ 미완료 항목 → 내일 스크럼 A단계에 자동 이월 제안
```

### 4-3. 태스크 (/tasks)

#### 메뉴 정의
> 나에게 배정된/내가 만든 모든 태스크를 한 곳에서 관리. 프로젝트와 무관하게 통합 뷰 제공.

#### 시나리오: 일반 사용자 — 내 태스크 확인

```
[일반] 사이드바 → 태스크
    │
    ├─ [All] 나에게 배정된 전체 태스크 (상태별 그룹)
    ├─ [Today] 오늘 마감 또는 스크럼에서 오늘 배정된 것
    ├─ [Upcoming] 다가오는 마감
    ├─ [Completed] 완료 아카이브
    └─ [My Tasks] 내가 직접 생성한 태스크

    → 태스크 클릭 → 사이드 패널에서 상세 확인/수정
    → 상태 변경: pending → in_progress → review → completed
```

#### 태스크 속성 (확정)

| 속성 | 값 | 용도 |
|---|---|---|
| **우선순위** | 긴급 / 높음 / 보통 / 낮음 | 업무 순서 결정 |
| **난이도** | 쉬움 / 보통 / 어려움 (3단계) | 업무 스트레스 측정용 |
| **예상 소요시간** | 시간 단위 (float) | 시간 배치 + 부하 계산 |
| **실제 소요시간** | 시간 단위 (float) | 퍼포먼스 분석 |
| **가중치** | 일반(1) / 핵심(2) / 마일스톤(3) | 진행률 계산 |

> **스트레스 지표 (향후 v0.6 퍼포먼스 모듈 연동 예정)**
> `업무 부하 = Σ(난이도 점수 × 예상시간 × 긴급도 가중치)` 로 팀원별 스트레스 수준 추적 가능.
> 난이도 점수: 쉬움=1, 보통=2, 어려움=3

#### 시나리오: 일반 사용자 — 태스크 직접 생성

```
[일반] 태스크 페이지 → "+ 새 태스크" 클릭
    │
    ├─ 태스크명 입력
    ├─ 프로젝트 연결 (선택 — 프로젝트 없이도 생성 가능)
    ├─ 마감일 설정
    ├─ 우선순위 선택 (긴급/높음/보통/낮음)
    ├─ 난이도 선택 (쉬움/보통/어려움)
    ├─ 예상 소요시간 입력
    ├─ 담당자 (기본: 본인)
    └─ 생성 완료

    → 프로젝트에 연결했으면 프로젝트 태스크 목록에도 표시
    → 연결 안 했으면 개인 태스크로만 존재
```

#### 시나리오: PM — 팀원에게 태스크 배정

```

    │
    ├─ 태스크명 + 상세 설명 입력
    ├─ 프로젝트 연결 (필수: 어떤 프로젝트의 태스크인지)
    ├─ 담당자 선택 (팀원 검색/선택)

```

    ├─ 컬럼: 대기 | 진행중 | 검토 | 완료 | 이슈
    ├─ 태스크 카드를 드래그로 상태 변경
    ├─ 카드에 담당자 아바타 + 마감일 + 우선순위 표시

#### 시나리오: 태스크 상태 흐름

```
[생성] → pending (대기)
    │
    ▼ 담당자가 시작
[진행] → in_progress (진행중)
    │
    ├─ 문제 발생 → issue (이슈)
    │   └─ 해결 후 → in_progress로 복귀
    │
    ├─ 보류 필요 → on_hold (보류)
    │   └─ 재개 시 → in_progress로 복귀
    │
    ▼ 작업 완료, 검토 요청
[검토] → review (검토)
    │
    ├─ 수정 필요 → in_progress로 복귀
    │
    ▼ 검토 통과
[완료] → completed
```

#### 시나리오: 태스크 보류 (on_hold)

```
[태스크 보류 — 2가지 경우]

[Case 1] 프로젝트 보류에 의한 자동 보류
    │
    ├─ PM이 프로젝트를 on_hold로 변경
    ├─ 해당 프로젝트의 진행중(in_progress) 태스크 → 자동으로 on_hold
    ├─ 대기(pending) 태스크 → 상태 유지 (원래 시작 전)
    ├─ 담당자에게 "프로젝트 보류로 태스크가 일시정지되었습니다" 알림
    └─ 프로젝트 재개 시 → 보류된 태스크 자동으로 in_progress 복귀

[Case 2] 개별 태스크 보류
    │
    ├─ 담당자 또는 PM이 태스크를 "보류"로 변경
    ├─ 보류 사유 입력 (필수): "자료 대기 중", "클라이언트 피드백 대기" 등
    ├─ 예상 재개일 입력 (선택)
    │
    ▼
[결과]
    · 태스크 상태: on_hold
    · 대시보드/스크럼에서 "보류 중" 별도 표시 (진행중과 구분)
    · 보류 기간은 실제 소요시간에서 제외
    · 재개: 담당자 또는 PM이 "재개" 클릭 → in_progress 복귀
```

[피드백 흐름 예시]
    │
    │  PM: "디자인 초안 확인해주세요" + [시안_v1.png 첨부]

    │

[알림]
    · 새 코멘트 → 태스크 담당자 + @멘션된 사람에게 알림
    · 첨부파일 추가 시에도 동일 알림
```

#### 시나리오: 결과보고 (셀프 프로젝트 전용)

[셀프 프로젝트 완료 시]
    │
    ▼
[결과보고 작성 화면]
    │
    ├─ 제목 (자동: 프로젝트명 + "결과보고")
    ├─ 요약: 뭘 했는지 (1~3줄)
    ├─ 결과물: 링크 또는 파일 첨부
    ├─ 배운 점 / 개선점 (선택)
    │
    ▼
[공유]
    · "팀 전체에 공유" 버튼 클릭
    · 팀 피드(대시보드 또는 별도 공유 페이지)에 게시
    · 전 팀원에게 알림: "김성현님이 '디자인 시스템 정리' 결과를 공유했습니다"
```

#### 시나리오: 프로젝트 삭제

```
[프로젝트 삭제 — 권한자만 가능]

[삭제 권한]
    · PM: 본인이 생성한 프로젝트만 삭제 가능
    · 대표: 모든 프로젝트 삭제 가능
    · 관리자/일반: 삭제 불가

[PM] 프로젝트 상세 → 설정 → "프로젝트 삭제"
    │
    ├─ 확인 다이얼로그: "이 프로젝트와 관련된 모든 태스크가 함께 삭제됩니다"
    ├─ 프로젝트명 재입력 확인 (실수 방지)
    │
    ▼
[삭제 조건]
    · 진행중(active) 태스크가 있으면 경고: "진행중 태스크 n건이 있습니다"
    · 강제 삭제 가능 (사유 입력)
    │
    ▼
[결과]
    · 프로젝트 + 연관 태스크 소프트 삭제 (DB에는 보관, UI에서 숨김)
    · 팀원에게 "프로젝트가 삭제되었습니다" 알림
    · 30일간 복구 가능 (설정 → 휴지통)
    · 30일 후 영구 삭제
```

#### 시나리오: 프로젝트 아카이브

```
[프로젝트 아카이브 — 완료된 프로젝트 정리용]

[완료된 프로젝트] → 자동 아카이브 대상
    │
    ├─ 완료 후 30일 경과 → "아카이브하시겠습니까?" 알림 (PM에게)
    ├─ PM이 수동으로 "아카이브" 클릭 가능
    │
    ▼
[결과]
    · 프로젝트 상태: archived
    · 프로젝트 목록에서 기본 숨김 (필터: "아카이브 포함" 체크 시 표시)
    · 읽기 전용 — 태스크/코멘트 수정 불가
    · 아카이브 해제: PM 또는 대표가 "아카이브 해제" → active/completed 복귀
```

### 4-5. 킥오프 (/projects/[id]/kickoff)

#### 메뉴 정의
> 프로젝트 유형 중 "프로젝트"에서만 필수. 프로젝트 상세의 탭으로 접근.
> **킥오프 완료 전에는 태스크 생성 불가** — 절차 강제.
> 체크리스트(시작 전 확인) + 미팅 기록(킥오프 회의 내용) 두 가지 역할을 담당.

#### 킥오프 구성

| 섹션 | 내용 | 성격 |
|---|---|---|
| **프로젝트 개요** | 목표, 범위, 제약조건, 성공기준 | 체크리스트 |
| **팀 구성** | R&R 배정 확인, 역할별 담당자 | 체크리스트 |
| **마일스톤** | 주요 일정 타임라인 확인 | 체크리스트 |
| **체크리스트** | 시작 전 확인 항목 (담당자별 체크) | 체크리스트 |
| **킥오프 노트** | 킥오프 미팅 내용 기록 (자유 텍스트) | 미팅 기록 |
| **합의 사항** | 미팅에서 합의된 핵심 결정 사항 | 미팅 기록 |
| **리스크/이슈** | 미팅에서 식별된 리스크와 대응 방안 | 미팅 기록 |

#### 시나리오: PM이 킥오프를 진행한다

```
[PM] 프로젝트 생성 완료 → 킥오프 탭 진입
    │
    ▼
[Step 1] 프로젝트 개요 작성
    │  · 목표: "2026 여름 브랜딩 캠페인 영상 3편 제작"
    │  · 범위: "영상 기획/촬영/편집, 디자인 썸네일"
    │  · 제약조건: "예산 500만원, 납기 03/27"
    │  · 성공기준: "영상 조회수 1만+ 달성"
    │
    ▼
[Step 2] 팀 구성 확인
    │  · 프로젝트 생성 시 배정한 R&R이 자동으로 표시됨
    │  · PM이 최종 확인 / 수정 가능
    │  · 팀원별 "숙지 확인" 체크 요청
    │
    ▼
[Step 3] 마일스톤 확인
    │  · 프로젝트 생성 시 설정한 마일스톤 표시
    │  · 수정/추가 가능
    │
    ▼
[Step 4] 체크리스트 확인
    │  · 항목별 담당자가 체크
    │  · 예: ☑ 레퍼런스 수집 완료 (박지원)
    │       ☑ 촬영 장비 확인 (이준호)
    │       ☐ 스크립트 초안 (박지원)
    │  · PM이 항목 추가/삭제/순서 변경 가능
    │
    ▼
[Step 5] 킥오프 미팅 진행 후 기록
    │  · 킥오프 노트: 미팅 내용 자유 기록
    │  · 합의 사항: "영상 톤은 밝고 캐주얼하게"
    │  · 리스크: "야외 촬영 시 우천 대비 실내 대안 필요"
    │
    ▼
[Step 6] 킥오프 완료 처리
    │  · PM이 "킥오프 완료" 버튼 클릭
    │  · 필수 항목 미체크 시 경고: "체크리스트 3건 미완료"
    │  · 강제 완료 옵션: "미완료 항목 있지만 진행" (사유 입력)
    │
    ▼
[결과]
    · 킥오프 상태: completed
    · ★ 태스크 생성 가능해짐 (이전까지 태스크 탭에 "킥오프를 먼저 완료하세요" 표시)
    · 팀원에게 "킥오프 완료, 태스크가 배정됩니다" 알림
```

#### 시나리오: 팀원이 킥오프를 확인한다

```
[팀원] 알림 수신: "새 프로젝트에 배정되었습니다"
    │
    ▼
[프로젝트 상세 → 킥오프 탭]
    │
    ├─ 프로젝트 개요 읽기 (목표/범위/제약조건)
    ├─ 내 역할 확인 (R&R에서 내가 어떤 역할인지)
    ├─ 마일스톤 확인 (내가 관련된 마감일)
    ├─ 체크리스트에서 내 담당 항목 체크
    │   예: ☑ "촬영 장비 확인" ← 이준호가 체크
    └─ "숙지 확인" 버튼 클릭
        → PM에게 "이준호님이 킥오프를 숙지했습니다" 알림
```

#### 시나리오: 킥오프 미완료 상태에서 태스크 생성 시도

```
[PM 또는 팀원] 프로젝트 상세 → 태스크 탭
    │
    ▼
[태스크 생성 차단]
    · "킥오프를 먼저 완료해주세요" 메시지 표시
    · "킥오프로 이동" 버튼 제공
    · 태스크 목록은 빈 상태, 생성 버튼 비활성화
```

#### 킥오프 vs 미니프로젝트/셀프 프로젝트

| 유형 | 킥오프 | 태스크 생성 조건 |
|---|---|---|
| **미니프로젝트** | ❌ 없음 | 프로젝트 생성 즉시 가능 |
| **프로젝트** | ✅ 필수 | 킥오프 완료 후 가능 |
| **셀프 프로젝트** | ❌ 없음 | 프로젝트 생성 즉시 가능 |

### 4-6. 일정 (/schedule)

#### 메뉴 정의
> 회사 전체 일정을 관리하는 캘린더. 전 직원 동일 열람+생성 권한.

#### 시나리오: 일정 확인 (전 역할 공통)

```
[사용자] 사이드바 → 일정
    │
    ├─ 월간 뷰 (기본): 이번 달 전체 일정
    ├─ 주간 뷰: 이번 주 시간대별 일정
    ├─ 일간 뷰: 오늘 상세 일정
    │
    ├─ 카테고리 필터 (사이드바)
    │   · 휴가 (그린)
    │   · 연봉협상 (오렌지)
    │   · 생일 (핑크)
    │   · 공휴일 (레드)
    │   · 회사 일정 (브랜드 옐로)
    │   · 회의 (블루)
    │   · 기타 (그레이)
    │
    └─ 일정 클릭 → 팝오버로 상세 확인
```

#### 시나리오: 일정 생성

```
[사용자] 캘린더에서 날짜 클릭 → 일정 생성 다이얼로그
    │
    ├─ 일정명
    ├─ 카테고리 선택 (색상 자동 적용)
    ├─ 시작일/종료일 (또는 종일)
    ├─ 시작 시간/종료 시간 (종일이 아닐 때)
    ├─ 반복 설정: 없음 / 매년 / 매월 / 매주
    ├─ 전사 공개 여부
    └─ 생성 완료 → 캘린더에 표시
```

#### 시나리오: 일정 수정/삭제 권한

```
[일정 수정/삭제]
    │
    ├─ 본인이 만든 일정: 수정/삭제 가능
    ├─ 타인이 만든 일정: 열람만 가능
    ├─ 관리자/대표: 모든 일정 수정/삭제 가능
    └─ 공휴일 (시스템 등록): 관리자만 수정 가능
```

#### 시나리오: 데일리 스크럼과 연동

```
[사용자] 데일리 스크럼 C단계 (시간 배치)
    │
    ├─ 오늘의 일정이 타임라인에 자동 블록으로 표시
    │   예: "10:00~11:00 주간회의" → 회색 블록 (수정 불가)
    ├─ 태스크를 회의와 겹치지 않게 배치
    └─ 일정 메뉴와 데이터 동기화
```

    │   · "브랜딩 가이드 업데이트" — 완료

### 4-9. AI 리뷰

#### 메뉴 정의
> 프로젝트/태스크의 완성된 결과물을 AI가 자동 리뷰하는 기능.
> 기존에 수동으로 하던 작업 (폼+태스크 리스트 → md로 정리 → AI에게 질문)을 **버튼 하나**로 자동화.

#### 시나리오: PM이 프로젝트 AI 리뷰 요청

```
[PM] 프로젝트 상세 → "AI 리뷰" 버튼 클릭
    │
    ▼
[자동 수집] 시스템이 아래 데이터를 md 형식으로 자동 생성
    │  · 프로젝트 기본 정보 (유형, 목표, 범위, 기간)
    │  · 킥오프 내용 (프로젝트 유형인 경우)
    │  · 태스크 전체 목록 (상태, 담당자, 마감일, 난이도, 소요시간)
    │  · 마일스톤 달성 현황
    │  · 지연된 태스크/이슈 목록
    │
    ▼
[AI 리뷰 실행]
    │  · 수집된 md를 AI(Gemini)에 전달
    │  · AI가 분석 후 리뷰 리포트 생성
    │
    ▼
[리뷰 결과 표시]
    │
    ├─ 📊 전체 진행률 요약
    ├─ ⚠️ 리스크 포인트 (지연 가능성, 병목 태스크)
    ├─ 💡 개선 제안 (리소스 재배치, 우선순위 조정 등)
    ├─ 📋 태스크별 상태 분석
    └─ 🎯 다음 단계 추천

    → 리뷰 결과를 프로젝트 상세에 저장 (이력 관리)
    → "팀에 공유" 버튼으로 팀원에게 전달 가능
```

#### 시나리오: 일반 사용자가 태스크 AI 리뷰 요청

```
[담당자] 태스크 상세 → "AI 리뷰" 버튼 클릭
    │
    ▼
[자동 수집]
    │  · 태스크 정보 (이름, 설명, 난이도, 예상/실제 소요시간)
    │  · 코멘트 이력 (피드백 내용)
    │  · 관련 프로젝트 컨텍스트
    │
    ▼
[AI 리뷰 결과]
    │
    ├─ 완료도 평가 (설명 대비 실제 작업 범위)
    ├─ 시간 분석 (예상 vs 실제, 효율성)
    ├─ 개선 포인트
    └─ 다음 액션 제안

    → 결과는 태스크 코멘트로 자동 추가 (AI 리뷰 태그)
```

#### AI 리뷰 권한

| 역할 | 프로젝트 리뷰 | 태스크 리뷰 |
|---|---|---|
| **대표** | 모든 프로젝트 | 본인 태스크 |
| **관리자** | 모든 프로젝트 | 본인 태스크 |
| **PM** | 담당 프로젝트 | 본인 + 담당 프로젝트 태스크 |
| **일반** | ❌ | 본인 태스크만 |

### 4-10. 회의록 → 태스크 연동 (action_items = 태스크)

#### 정의
> 회의록에서 나온 액션아이템(action_items)은 곧 태스크(Task)임.
> 회의록 모듈에서 액션아이템을 작성하면 태스크로 자동 생성/연결됨.

#### 시나리오: 회의록에서 액션아이템 → 태스크 자동 생성

```
[PM/사용자] 회의록 작성 중 → 액션아이템 추가
    │
    ├─ 액션아이템 입력
    │   · 내용: "홈페이지 메인 시안 제작"
    │   · 담당자: 박지원
    │   · 마감일: 03/15
    │
    ▼
[저장 시 자동 처리]
    │
    ├─ 태스크 자동 생성
    │   · 태스크명 = 액션아이템 내용
    │   · 담당자 = 지정된 담당자
    │   · 마감일 = 지정된 마감일
    │   · 출처: "회의록 — 2026-03-06 주간회의" (링크)
    │   · 프로젝트 연결: 회의록에 프로젝트가 연결된 경우 자동 매핑
    │
    ├─ 담당자에게 알림: "회의록에서 새 태스크가 배정되었습니다"
    │
    └─ 회의록 내 액션아이템 옆에 태스크 링크 표시
        → 클릭 시 태스크 상세로 이동
```

#### 시나리오: 태스크에서 회의록 출처 확인

```
[담당자] 태스크 상세 확인
    │
    ├─ 출처 표시: "📋 회의록 — 2026-03-06 주간회의"
    │   → 클릭 시 회의록으로 이동
    │
    └─ 태스크 상태 변경 시 → 회의록 액션아이템 상태도 자동 동기화
        · 태스크 completed → 액션아이템에 ✅ 표시
        · 태스크 in_progress → 액션아이템에 🔄 표시
```

#### 연동 범위

| 항목 | 설명 |
|---|---|
| **생성 방향** | 회의록 → 태스크 (단방향 자동 생성) |
| **상태 동기화** | 태스크 상태 변경 → 액션아이템 상태 자동 반영 (양방향) |
| **삭제 처리** | 태스크 삭제 시 액션아이템 링크 해제 (액션아이템은 유지) |
| **프로젝트 연결** | 회의록에 프로젝트가 연결되어 있으면 태스크도 해당 프로젝트에 자동 배정 |
