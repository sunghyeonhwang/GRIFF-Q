# GRIFF-Q 사내 업무 관리 시스템 기획서

> **문서 버전**: v1.0
> **프로젝트명**: GRIFF-Q
> **목적**: Asana / Slack / Google Sheets로 분산된 업무를 하나의 내부 웹 시스템으로 통합

---

## 1. 프로젝트 개요

### 1-1. 배경

현재 팀 업무가 Asana(프로젝트 관리), Slack(소통), Google Sheets(견적서/회의록/입금 관리) 등 여러 도구에 분산되어 있어 정보 파편화, 이중 입력, 현황 파악 지연 문제가 반복 발생하고 있다. GRIFF-Q는 이를 하나의 통합 시스템으로 해결한다.

### 1-2. 핵심 가치

| 가치 | 설명 |
|---|---|
| **통합** | 분산된 업무 데이터를 하나의 시스템으로 통합 |
| **효율** | 반복 작업(견적서 복사, 수동 집계 등) 자동화 |
| **가시성** | 대시보드를 통한 전체 업무 현황 실시간 파악 |
| **추적성** | 모든 변경 이력의 감사 로그 기록 |

### 1-3. 대상 사용자

- 팀원 7명 (본인 포함)
- 역할: 온라인 디자이너 2명, 오프라인 디자이너 1명, 기획자 2명, 영상 콘텐츠 개발자 1명, 올라운더 1명

## 2. 기술 스택

| 영역 | 기술 | 비고 |

## 3. 권한 체계

### 3-1. 역할 4단계

```
super > boss > manager > normal
```

| 역할 | 설명 |
|---|---|
| `super` | 전체 시스템 관리, 계정/권한 변경, 롤백 |
| `boss` | 대부분 관리 권한, 계정 관리 접근 |
| `manager` | CRUD 전체 가능, 설정 일부 제한 |
| `normal` | 읽기 + 본인 데이터 작성만 가능 |

### 3-2. 페이지별 접근 권한

| 페이지 | super | boss | manager | normal |
|---|:---:|:---:|:---:|:---:|
| 대시보드 | 전체 | 전체 | 전체 | 전체 |
| 회고 - 개인 작성 | 전체 | 전체 | 전체 | 전체 |
| 회고 - 취합 뷰 | 전체 | 전체 | 전체 | 읽기만 |
| 회의록 | 전체 | 전체 | 전체 | 읽기만 |
| 입금/결제 요청 | 전체 | 전체 | 전체 | 읽기만 |
| 견적서 | 전체 | 전체 | 전체 | 읽기만 |
| 클라이언트 예측 | 전체 | 전체 | 전체 | 접근 불가 |
| 설정 - 계정 관리 | 전체 | 전체 | 접근 불가 | 접근 불가 |
| 설정 - 변경 이력 | 전체 | 전체 | 읽기만 | 접근 불가 |

## 5. 기능 모듈 상세

### 5-1. 인증 (Auth)

| 항목 | 내용 |
|---|---|
| **인증 방식** | 이메일 + 비밀번호 (Supabase Auth) |
| **세션** | 쿠키 기반 세션, 미들웨어에서 자동 체크 |
| **권한** | public.users 테이블에 role 컬럼, RLS 정책으로 행 단위 접근 제어 |
| **리다이렉트** | 미인증 → /login, 권한 부족 → 대시보드 |

### 5-2. 대시보드 (`/dashboard`)

로그인 후 첫 화면. 전체 업무 현황을 한눈에 파악한다.

**상단 영역**
- 인사말: "안녕하세요, {이름}님" + 오늘 날짜/요일
- 퀵 액션 버튼: "새 회의록", "새 견적서", "입금 등록"

**요약 카드 (4종)**
| 카드 | 데이터 소스 |
|---|---|
| 마감 임박 | 7일 이내 미완료 액션아이템 수 |
| 입금 대기 | 대기 상태 결제 요청 수 |
| 최근 회의 | 7일 이내 회의록 수 |
| 프로젝트 | 활성 프로젝트 수 |

**차트 위젯 (3종)**
| 차트 | 타입 | 데이터 |
|---|---|---|
| 월별 입금 추이 | AreaChart | 최근 6개월 입금 완료 금액 |
| 견적 상태 분포 | PieChart | 작성중/확정/발송완료 비율 |
| 회고 제출률 | BarChart | 프로젝트별 제출/미제출 현황 |

**하단 위젯**
- 마감 임박 액션아이템 테이블
- 대기 중인 입금 요청
- 최근 회의록 (카드 링크)
- 최근 활동 타임라인 (audit_logs 기반)

**레이아웃**: Bento 그리드 (반응형 — 모바일 1열, 데스크톱 2~3열)

### 5-3. 회고 시스템 (`/retrospective`)

프로젝트 팀 회고를 위한 개인 작성 + 취합/정리 시스템.

#### 적용 프레임워크
- **KPT**: Keep / Problem / Try
- **SSC**: Start / Stop / Continue
- 두 프레임워크를 혼합하여 사용

#### 개인 회고 작성 (`/retrospective/new`)

**입력 필드**
| 필드 | 타입 | 비고 |
|---|---|---|
| 프로젝트명 | Select (기존 목록) + 직접 입력 | 새 프로젝트 생성 다이얼로그 |
| 회고 기간 | DateRangePicker | |
| 작성자 | 자동 채움 (로그인 사용자) | |
| 역할 | 복수 선택 Checkbox | 디자인, 인쇄, 웹사이트 제작, 브랜딩, 촬영, 중계, 라이브 중계, 사진촬영, 진행, 홍보, PR (11종) |
| KPT 섹션 | Textarea × 3 | Keep / Problem / Try — 항목 동적 추가/삭제 |
| SSC 섹션 | Textarea × 3 | Start / Stop / Continue — 항목 동적 추가/삭제 |
| 만족도 점수 | 1~10 슬라이더 | |
| 파트별 점수 | 카테고리별 1~10 | |
| 종합 의견 | Textarea | |
| 공유 메모 | Textarea (선택) | 팀 공유 포인트 + 다음 반영 사항 |

**동작 규칙**
- 임시 저장(Draft) 지원
- 제출(Submitted) 후 수정 불가 (super/boss만 예외)

#### 프로젝트별 취합 뷰 (`/retrospective/summary/[projectId]`)

**탭 구성 (5개)**
| 탭 | 내용 |
|---|---|
| 만족도 | 팀 평균 점수 + 개인별 점수 분포 |
| 파트별 | 카테고리별 평균 점수 비교 |
| KPT + SSC | Keep+Continue / Problem+Stop / Try+Start 자동 묶기 |
| 종합 교훈 | 최종 합의 사항 편집 (manager 이상) |
| 참여 현황 | 전체 사용자 대비 제출 현황 테이블 |

#### 포스트모템 (`/retrospective/postmortem`)

**타임라인 회고 구조**
- 각 타임라인 항목: 날짜+시간 / 제목 / 설명 / 근본 원인 / 교훈 / 액션아이템
- Accordion UI (접기/펼치기)
- 색상 코딩: 근본 원인(빨강) / 교훈(파랑) / 액션아이템(초록)
- 심각도 배지: Critical / Major / Minor

### 5-4. 회의록 관리 (`/meetings`)

#### 목록 (`/meetings`)
- 테이블 뷰: 제목, 날짜, 참석자, 액션아이템 완료율
- 검색: 키워드 (제목/내용)
- 필터: 날짜 범위 (7일/30일/90일/전체)
- 정렬: 최신순 / 오래된순 / 미완료 액션아이템 우선

#### 작성/수정 (`/meetings/new`, `/meetings/[id]`)

**입력 필드**
| 필드 | 타입 |
|---|---|
| 회의 제목 | Input |
| 회의 날짜 | DatePicker |
| 참석자 | MultiSelect (팀원 목록) |
| 회의 내용 | Textarea |
| 액션아이템 | 인라인 편집 테이블 |

**액션아이템 테이블 컬럼**
- 항목명 / 담당자(Select) / 마감일(DatePicker) / 상태(대기/진행중/완료) / 비고

#### Google Sheets 파싱 입력 (`/meetings/import`)
- 텍스트 영역에 Sheets 내용 붙여넣기 → 탭/줄바꿈 자동 파싱 → 미리보기 → 저장

#### 액션아이템 대시보드 (`/meetings/action-items`)
- 전체 액션아이템 통합 뷰
- 담당자 필터
- 진행률 시각화 (Progress 바)

#### 퍼블리시
- Excel 다운로드 (회의록 + 액션아이템 2시트)
- Google Sheets 내보내기
- 인쇄용 레이아웃 (`@media print` 최적화)

### 5-5. 입금/결제 요청 (`/payments`)

#### 목록 (`/payments`)
- 요약 카드 3종: 대기 중 건수 / 대기 총액 / 완료 건수
- **기본 뷰**: 은행명 기준 그룹핑 카드 (은행별 Card + Table)
- **대체 뷰**: 전체 테이블 뷰
- 뷰 전환 토글 (테이블 ↔ 카드)
- 상태 필터: 전체 / 대기 / 완료
- 마감일 임박순 정렬
- 복사 기능: 계좌번호/금액 클립보드 복사 버튼

#### 등록/수정 (`/payments/new`, `/payments/[id]`)

**입력 필드**
| 필드 | 타입 | 비고 |
|---|---|---|
| 이름 | Input | |
| 금액 | Input (숫자) | 천단위 콤마 자동 포맷 |
| 은행 | Select | 19개 은행 프리셋 |
| 계좌번호 | Input | |
| 입금자명 | Input | |
| 마감일 | DatePicker | |
| 비고 | Textarea | |

**동작**: 완료 처리 버튼 (입금 확인), 완료 상태에서 super/boss만 수정 가능

#### 퍼블리시
- Excel 내보내기 (전체 입금 내역)

#### 향후 확장 (v0.3)

- 계산서 발행 조회: 외부 API 연동

### 5-6. 견적서 공동 작업 (`/estimates`)

#### 목록 (`/estimates`)
- 요약 카드 3종: 작성중 / 확정 / 발송완료 건수
- 테이블 뷰: 프로젝트명, 클라이언트, 견적일, 상태, 편집 잠금, 작성자
- 카드 뷰 토글 가능

#### 견적서 편집 — 심플 모드 (`/estimates/[id]/edit`)

**기본 정보**
- 프로젝트명, 클라이언트명, 견적일, 유효기간, 비고

**항목 테이블 (순차 편집 + 잠금)**
| 컬럼 | 설명 |
|---|---|
| 항목명 | 텍스트 입력 |
| 수량 | 숫자 |
| 단가 | 숫자 (천단위 콤마 자동) |
| 금액 | 자동 계산 (수량 × 단가) |
| 비고 | 텍스트 |
| 하이라이트 | 셀 배경색 지정 (변경/주의 항목 강조) |

**자동 계산**: 소계 / 부가세(10%) / 총액

**편집 잠금 방식**
- 편집 시작 시 다른 사용자에게 "편집 중" 표시
- 잠금 자동 해제: 타임아웃 5분

#### 견적서 편집 — 실시간 동시 편집 모드 (`/estimates/collab/[id]`)

**Supabase Realtime 기반**
- `estimate:{id}` 채널 구독
- 셀 단위 debounce 저장 (300ms)
- 충돌 해결: Last Writer Wins + 변경 알림 Toast

**Presence UI**
- 상단 접속자 아바타 표시 (최대 5명 + "+N")
- 각 사용자의 편집 셀 위치 표시 (셀 테두리 색상)
- 동일 셀 동시 편집 감지 → Toast 알림

#### 작년 견적 비교 (`/estimates/[id]/compare`)
- 좌우 분할 비교 뷰 (현재 vs 작년)
- 항목별 금액 차이 색상 코딩 (증가: 빨강 / 감소: 초록)
- 새로 추가/삭제된 항목 하이라이트

#### 견적 템플릿 (`/estimates/templates`)
- 자주 사용하는 항목 세트를 템플릿으로 저장
- 견적서 생성 시 템플릿 선택 → 항목 자동 채움

#### 퍼블리시
- PDF 다운로드 (`@react-pdf/renderer`, 한글 폰트, A4 레이아웃)
- Excel 다운로드 (견적정보 + 견적항목 2시트)
- Google Sheets 내보내기 (Service Account 방식)

### 5-7. 클라이언트 반응 예측 (`/predict`)

> 클라이언트별 페르소나 아바타를 생성하고, AI를 통해 예상 반응을 시뮬레이션하는 시스템.

#### 아바타 목록 (`/predict/avatars`)
- 카드 그리드: 아바타 아이콘, 이름, 소속, 성격 키워드 배지
- 설정/채팅 바로가기

#### 아바타 생성 & 인격 설정

**기본 정보**
| 필드 | 타입 |
|---|---|
| 아바타 이름 | Input (예: 김과장, 이대리) |
| 소속/회사명 | Input |
| 직급/역할 | Input |
| 아바타 이미지 | 기본 제공 아이콘 8종 선택 |

**수동 인격 프로필**
| 필드 | 타입 |
|---|---|
| 말투 스타일 | Select (격식체/비격식체, 직설적/완곡, 간결/장문) |
| 성격 키워드 | 태그 입력 (예: 꼼꼼함, 가격 민감, 빠른 의사결정) |
| 의사결정 패턴 | Select (즉답형 / 검토 후 답변형 / 상사 확인형) |
| 민감 주제 | 태그 입력 (예: 예산, 일정 지연) |
| 추가 메모 | Textarea |

#### 대화 데이터 학습 (`/predict/avatars/[id]/upload`)
- 지원 형식: `.txt` (카카오톡 내보내기), `.csv` (이메일 대화)
- Gemini API로 분석 → 말투/성격/패턴/민감주제 자동 추출
- 추출된 프로필을 아바타 인격에 자동 반영 (사용자 확인 후)

#### 예측 채팅 (`/predict/chat/[avatarId]`)

**기본 채팅**
- 2패널 레이아웃: 세션 목록(좌) + 채팅(우)
- 사용자 입력(우리 측 발언) → 아바타 AI 응답 (Gemini API)
- 시스템 프롬프트에 인격 프로필 자동 주입
- 대화 이력 저장/불러오기

**감정 분석 패널**
- 긍정/중립/부정 3단계 게이지 바
- 민감 주제 감지 시 경고 배지
- 부정 감정 감지 → "이렇게 말해보세요" 대안 3가지 제안

**상황 시뮬레이션 모드**
- 프리셋 시나리오: 견적 네고 / 일정 변경 요청 / 불만 접수 / 추가 요청 / 커스텀
- 시나리오 선택 → 시스템 프롬프트에 상황 컨텍스트 주입

**대화 내보내기**: 텍스트/PDF 다운로드 + Gemini 요약 생성

### 5-9. 공통 기능

#### 인앱 알림 시스템
- 헤더 알림 벨 아이콘 + 읽지 않은 건수 배지
- 최근 20건 드롭다운, 클릭 시 해당 페이지 이동 + 읽음 처리

- 300ms 디바운스, 카테고리별 5건 제한

#### 모바일 반응형
- 모든 페이지 모바일(375px~) 대응
- 테이블 → 카드 뷰 자동 전환 (모바일에서 카드 기본)
- 페이지 헤더: `flex-col sm:flex-row`
- 테이블: 가로 스크롤 처리

|---|---|

### D-5. 보관위치 관리

#### 위치 체계 표준

```
센터 > 구역 > 랙 > 선반 > 칸

예시: STUDIO-A > A구역 > R02 > S03 > B04
위치코드: A-R02-S03-B04 (고유)
```

#### 위치 마스터 정보
- 필수 항목: 위치코드, 위치명, 상위위치코드, 위치타입, 수용량, 활성여부
- 위치타입: 창고 / 랙 / 선반 / 촬영장 / 임시보관 / 정비구역 / 외부반출
- 트리 구조로 상하위 위치 탐색 지원

#### 자산 이동 처리 규칙
- 자산은 항상 1개의 현재 위치만 유지
- 이동 절차: 출발 위치 스캔 → 자산 QR 스캔 → 도착 위치 스캔
- 이동 이력 자동 저장: 담당자, 시각, 출발/도착 위치, 사유

#### 상태별 배치 제약

| 상태 | 제약 |
|---|---|
| 정비중 | 정비구역에만 배치 가능 |
| 대여중 | 외부위치(프로젝트/담당자)로 표시 |
| 분실 | 마지막 위치 및 마지막 스캔 시각 고정 저장 |

#### 위치 관리 화면 (운영 필수)
1. 위치 맵 뷰 (트리/평면)
2. 빈 위치 찾기 (수용량 기준)
3. 위치별 재고 목록
4. 최근 이동 로그
5. 세트 누락 탐지

#### 알림 정책
- 위치 미확정 자산 24시간 경과 → 알림
- 금지 위치 배치 → 즉시 경고
- 장기 미이동 자산 (60일) → 점검 알림

## 부록 J. 데일리 스크럼 상세 기획 (v0.4+)

### J-1. 핵심 컨셉

> 매일 아침 AI 대화형 인터페이스로 **"오늘 하루를 설계"**하는 시스템.
> 단순 보고가 아니라, 생각을 정리하고 → 우선순위를 잡고 → 시간에 배치하는 **완성된 플로우**를 거쳐야 스크럼이 완료된다.

### J-2. 3단계 완성 플로우

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  A. 브레인스토밍        B. 우선순위 정하기        C. 시간 배치하기  │
│  ──────────────      ───────────────       ──────────────  │
│  "오늘 뭘 해야 하지?"  "뭐가 더 급하지?"     "언제 하지?"       │
│                                                         │
│  ① AI 대화로 수집      ③ 드래그 정렬 /        ⑤ 타임블록 배치   │
│  ② 자유 입력/이월      AI 제안 / 수동 설정    ⑥ 예상 소요시간   │
│                       ④ 긴급/중요 매트릭스      캘린더 연동      │
│                                                         │
│  ─────────────────────────────────────────────────────  │
│  3단계 모두 완료 → ✅ 스크럼 완성 → Task 자동 등록        │
│  미완료 시 → ⚠️ 대시보드에 "스크럼 미완료" 표시           │
└─────────────────────────────────────────────────────────┘
```

#### A단계: 브레인스토밍 — "오늘 해야 할 일을 생각"

| 항목 | 내용 |
|---|---|
| **AI 대화형 수집** | "오늘 뭘 해야 하나요?" → 자연어 대화로 할 일 목록 추출 |
| **자동 이월** | 어제 미완료 Task → 자동 제안 ("어제 이것도 남았는데, 오늘 할까요?") |
| **프로젝트 연동** | 진행 중인 프로젝트의 마감 임박 Task 자동 표시 |
| **자유 입력** | 대화 외에 직접 항목 추가도 가능 |
| **회의 일정 반영** | 오늘 예정 회의 자동 표시 (시간 블록 미리 잡기) |
| **결과물** | 정리되지 않은 할 일 목록 (Raw List) |

#### B단계: 우선순위 정하기 — "뭐가 더 급한지 정리"

| 항목 | 내용 |
|---|---|
| **AI 자동 제안** | 마감일/프로젝트 중요도/의존성 기반 우선순위 추천 |
| **수동 조정** | 드래그 앤 드롭으로 순서 변경 |
| **긴급/중요 매트릭스** | 아이젠하워 매트릭스 시각화 (선택적 뷰) |
| **라벨링** | 긴급(빨강) / 중요(주황) / 일반(회색) / 후순위(파랑) |
| **의존성 표시** | "이걸 먼저 해야 저걸 할 수 있음" 연결 |
| **결과물** | 우선순위가 정해진 할 일 목록 (Prioritized List) |

#### C단계: 시간 배치하기 — "언제 할지 배치"

| 항목 | 내용 |
|---|---|
| **타임블록** | 할 일을 시간대에 드래그 배치 (09:00~18:00 타임라인) |
| **예상 소요시간** | 각 Task에 예상 소요시간 입력 (15분/30분/1시간/2시간/반나절) |
| **자동 배치 제안** | AI가 우선순위 + 소요시간 + 회의 시간을 고려해 배치 추천 |
| **버퍼 타임** | 예기치 못한 업무를 위한 여유 시간 자동 확보 (설정 가능) |
| **과부하 경고** | 할당 시간 > 가용 시간 → "오늘 다 못 할 수 있어요" 경고 |
| **캘린더 뷰** | GRIFF 스케줄과 연동, 회의/일정과 겹침 방지 |
| **결과물** | 시간이 배치된 오늘의 일정 (Today's Schedule) |

### J-3. 완성 후 자동화

| 자동화 | 내용 |
|---|---|
| **Task 자동 등록** | 스크럼 완성 시 각 항목 → Task 모듈에 자동 등록/수정 |
| **액션아이템 연결** | 기존 프로젝트/회의록 액션아이템과 자동 매핑 |
| **임무카드 연동** | 프로젝트 임무카드에 오늘 작업분 자동 연결 |
| **대시보드 반영** | "오늘의 흐름" 위젯에 스크럼 결과 표시 |
| **구조화 저장** | 대화 내용 → 구조화 데이터로 자동 변환/저장 |

### J-4. 미응답/미완료 처리

| 상황 | 대응 |
|---|---|
| 스크럼 미시작 | 오전 리마인드 알림 (설정 가능 시간) |
| A단계만 완료 | "우선순위를 정해주세요" 리마인드 |
| B단계까지 완료 | "시간 배치만 하면 완성!" 리마인드 |
| 전체 미응답 | PM(올라운더)에게 에스컬레이션 알림 |
| 블로커 발견 | 블로커 내용 → PM에게 자동 전달 |

### J-5. 하루 마감 회고 (선택)

| 항목 | 내용 |
|---|---|
