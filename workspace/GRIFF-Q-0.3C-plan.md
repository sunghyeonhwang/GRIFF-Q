# GRIFF-Q v0.3C — 데일리 스크럼

> v0.3 핵심 업무 엔진의 **일일 업무 설계 Phase**
> **상위 계획서:** `GRIFF-Q-0.3-plan.md`
> **선행 의존:** v0.3A (TASK 모듈), v0.3B (GRIFF 스케줄 — 시간 배치 연동)

---

## 1) 목표

매일 아침 **"오늘 하루를 설계"**하는 AI 대화형 3단계 시스템을 구축한다.

```
A단계: 브레인스토밍 — "오늘 뭘 해야 하지?" (AI 대화 + 자동 이월)
     ↓
B단계: 우선순위 — "뭐가 더 급하지?" (드래그 정렬 + 아이젠하워 매트릭스)
     ↓
C단계: 시간 배치 — "언제 하지?" (타임블록 + 캘린더 연동)
     ↓
완료 → Task 자동 등록 + 대시보드 반영
```

## 7) 세부 설계

### A단계: 브레인스토밍

| 기능 | 설명 |
|---|---|
| **AI 대화형 수집** | "오늘 뭘 해야 하나요?" → 자연어로 할 일 추출 |
| **자동 이월** | 어제 미완료 Task → "오늘 이어서 할까요?" 제안 |
| **프로젝트 연동** | 마감 임박 Task 자동 표시 |
| **자유 입력** | AI 대화 없이 직접 항목 추가 |
| **회의 일정 반영** | 오늘 예정 회의(schedules) 자동 표시 |
| **결과물** | Raw List (정리되지 않은 목록) |

### B단계: 우선순위

| 기능 | 설명 |
|---|---|
| **AI 자동 제안** | 마감일/중요도/의존성 기반 우선순위 추천 |
| **드래그 정렬** | dnd-kit 순서 변경 |
| **아이젠하워 매트릭스** | 긴급/중요 2×2 시각화 (선택 뷰) |
| **라벨링** | 긴급(빨강) / 중요(주황) / 일반(회색) / 후순위(파랑) |
| **의존성 표시** | 선행 Task 연결 표시 |
| **결과물** | Prioritized List |

### C단계: 시간 배치

| 기능 | 설명 |
|---|---|
| **타임블록** | 09:00~18:00 타임라인에 드래그 배치 |
| **예상 소요시간** | 15분 / 30분 / 1시간 / 2시간 / 반나절 |
| **자동 배치 제안** | AI가 우선순위 + 소요시간 + 회의 고려 |
| **버퍼 타임** | 여유 시간 자동 확보 (설정 가능) |
| **과부하 경고** | 할당 > 가용 시간 → 경고 |
| **캘린더 연동** | GRIFF 스케줄 겹침 방지 |
| **결과물** | Today's Schedule |

### 완료 후 자동화

| 자동화 | 내용 |
|---|---|
| Task 자동 등록 | 스크럼 항목 → Task 생성/업데이트 |
| 액션아이템 연결 | 기존 프로젝트/회의록과 매핑 |
| 대시보드 반영 | "오늘의 흐름" 위젯에 결과 표시 |

### 미응답/미완료 처리

| 상황 | 대응 |
|---|---|
| 스크럼 미시작 | 오전 리마인드 알림 |
| A단계만 완료 | "우선순위를 정해주세요" 리마인드 |
| B단계까지 완료 | "시간 배치만 하면 완성!" 리마인드 |
| 전체 미응답 | PM에게 에스컬레이션 |

### 하루 마감 회고

| 항목 | 내용 |
|---|---|
| 완료 체크 | 계획 Task 중 완료/미완료 체크 |
| 미완료 사유 | 간단 사유 입력 |
| 자동 이월 | 미완료 → 내일 A단계 자동 제안 |
| 달성률 | 오늘 달성률 → 퍼포먼스 모듈(v0.6) 연동 |

## 9) 사용자 시나리오

> 구현 직전 상세 보강 예정. 현재는 골격만 기재.

### S-C1. 팀원이 아침 데일리 스크럼을 진행한다

**플로우:** 로그인 → 스크럼 배너 → A단계(AI 대화 + 이월 + 자유 입력) → B단계(드래그 정렬) → C단계(타임블록 + 스케줄 연동) → 완료 → Task 자동 생성

**엣지 케이스:**
- AI 없이 직접 입력만: A단계에서 "직접 입력 모드" 토글
- C단계 없이 B까지만: "부분 완료" 상태 허용
- 오후 스크럼 시작: 현재 시각 이후만 가용 시간
- 기존 Task 중복: AI가 서브태스크 연결 제안

### S-C2. 팀원이 하루를 마감한다

**플로우:** 대시보드 → 마감 회고 → 완료/미완료 체크 → 미완료 사유 → 이월 처리 → 상태 전파

**엣지 케이스:**
- 마감 회고 미실시: 다음 날 자동 이월
- 계획 외 완료 Task: "추가 완료 Task" 입력란
- 다중 프로젝트: 프로젝트별 그룹 표시

### S-C3. PM이 팀 스크럼 현황을 모니터링한다

**플로우:** 대시보드 위젯 → 팀원별 스크럼 상태 → 미시작 팀원 리마인드 발송
