| 그룹 | 작업 | 담당 에이전트 | 예상 | 동시 실행 |
|------|------|-------------|------|:---------:|
| **Group A** | D1 (권한) + D3 (사이드바+검색) | Agent-1 | 12h | ✅ |
| **Group B** | D2 (대시보드 위젯) + D4 (알림) | Agent-2 | 14h | ✅ |

## 7. DB 마이그레이션

### 7.1 `020_role_permissions.sql` (D1)

```sql
-- RLS 정책 강화: 프로젝트 역할별 접근
-- projects: normal은 본인 배정 프로젝트만
-- tasks: normal은 본인 태스크만
-- scrums: 관리자 이상은 전체 열람

-- 프로젝트 삭제: soft delete 컬럼 추가
ALTER TABLE projects ADD COLUMN IF NOT EXISTS deleted_at timestamptz;
ALTER TABLE projects ADD COLUMN IF NOT EXISTS deleted_by uuid REFERENCES auth.users(id);
```

### 7.2 `021_cross_module.sql` (D5)

```sql
-- 회의록 액션아이템 ↔ 태스크 연결
ALTER TABLE action_items ADD COLUMN IF NOT EXISTS linked_task_id uuid REFERENCES tasks(id) ON DELETE SET NULL;
CREATE INDEX idx_action_items_task ON action_items(linked_task_id);

-- 회고 ↔ 프로젝트 연결 (retrospectives 테이블에 project_id 추가)
ALTER TABLE retrospectives ADD COLUMN IF NOT EXISTS project_id uuid REFERENCES projects(id) ON DELETE SET NULL;
CREATE INDEX idx_retrospectives_project ON retrospectives(project_id);
```

### 10.2 테스트 시나리오

| 시나리오 | 검증 항목 |
|----------|----------|
| 권한: normal 사용자 | 본인 태스크/프로젝트만 표시, 삭제 버튼 없음 |
| 권한: manager 사용자 | 담당 프로젝트 열람, 태스크 관리 가능 |
| 권한: boss/super | 전사 현황 열람, 프로젝트 승인/삭제 가능 |
| 사이드바 접이식 | 3그룹 토글, 현재 메뉴 하이라이트 |
| 글로벌 검색 | Cmd+K 열림, 프로젝트/태스크/회의록 검색 |
| 대시보드 위젯 | 4종 위젯 데이터 정상, 바로가기 동작 |
| 알림: 태스크 배정 | 담당자에게 알림 생성, 벨 뱃지 표시 |
| 알림: 마감 임박 | D-2 태스크 알림, 읽음 처리 |
| 회의록→태스크 | 액션아이템 생성 → 태스크 자동 생성 |
| 태스크→액션아이템 | 태스크 완료 → 액션아이템 상태 동기화 |
| 회고→프로젝트 | 프로젝트 완료 → 회고 작성 유도 |
| 회귀: 스크럼 | v0.3C 전체 플로우 정상 |
| 회귀: 킥오프/일정 | v0.3B 전체 플로우 정상 |
| 회귀: 프로젝트/태스크 | v0.3A 전체 플로우 정상 |
| 모바일 375px | 모든 새 UI 반응형 정상 |
| 다크모드 | 전체 다크모드 정상 |
