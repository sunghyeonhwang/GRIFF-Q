1. **Design 문서 상세도**
   - 알림 함수의 "실제 구현 방식" (recipientIds[], taskLink)을 설계 단계에서 더 구체적으로 명시
   - 결과: 구현 후 Gap 발생 (시그니처 차이)

2. **P0 Gap 사전 검증**
   - 구현 전에 "어느 Server Action에서 트리거되는가"를 체크리스트로 만들어야 함
   - D4 알림 함수 중 2개 호출 누락은 사전에 발견 가능했음

3. **에이전트 간 핸드오버**
   - Agent-1, 2, 3 작업 완료 후 리더의 "merge + 통합 테스트" 단계가 더 체계적이어야 함
   - 병렬 작업의 장점을 더 활용 가능

4. **QA 시나리오 커버리지**
   - 브라우저 QA는 기능 동작만 확인 (DB 데이터 적정성 미검증)
   - 예: 알림 생성 후 DB에 실제로 저장되었는지 확인 필요

### 4.3 다음 버전에 적용할 사항 (To Apply Next Time)
