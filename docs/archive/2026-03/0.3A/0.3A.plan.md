## 1. Background & Problem

### 1.1 현재 상태
- v0.25까지 인터페이스 개선 완료 (Auth + 7개 모듈 39페이지)
- 프로젝트 모듈이 단순 CRUD 수준으로, 실질적 업무 관리 기능 부재
- Task 관리 시스템 미존재 — 회의록 액션아이템이 후속 추적 불가

### 1.2 해결할 문제
1. 프로젝트를 **업무 관리 허브**로 전면 재설계
2. 독립 Task 시스템 (Kanban + 필터 + 서브태스크)
3. 의존성 시각화 (DAG 맵 + 크리티컬 패스)
4. AI 기반 프로젝트 리뷰 (대화형 질의 + 자동 보고서)

---

## 2. Goals & Success Criteria

### 2.1 목표
기존 프로젝트 모듈을 **업무 관리 허브**로 전면 재설계하고,
독립 TASK 관리 시스템 + 의존성 맵(DAG) + AI 리뷰를 구축한다.

### 2.2 완료 기준

- [ ] 프로젝트 메뉴 활성화 + 6종 뷰 전환 동작
- [ ] 프로젝트 종류별 R&R + 임무카드 자동 발급
- [ ] Task CRUD + Kanban 보드 드래그 동작
- [ ] Task 필터 뷰 5종 (All/Today/Upcoming/Completed/My)
- [ ] 프로젝트 진행률 자동 계산
- [ ] 태스크 의존성 맵 (React Flow + DAG 순환 검증 + 상태 전파)
- [ ] 의존성 맵 진행 현황 대시보드
- [ ] AI 실시간 대화형 리뷰 (Gemini)
- [ ] AI 주간/월간 보고서 자동 생성
- [ ] 라이트/다크 x 모바일/데스크톱 4조합 정상
- [ ] 기존 기능 회귀 테스트 통과

## 9. User Scenarios

**플로우:** TASK 메뉴 -> My Tasks -> 상세 패널 -> 서브태스크 체크 -> 상태 전파

**플로우:** 금요일 자동 생성 -> 알림 -> 보고서 탭 -> 7개 섹션 확인

## 12. Risk & Mitigation

| 리스크 | 영향 | 대응 |
|--------|------|------|
| React Flow 번들 크기 | 초기 로딩 지연 | dynamic import + 코드 스플리팅 |
| DAG 순환 참조 | 무한 루프 | 연결 시점에 DFS 기반 사전 검증 |
| Gemini API 비용 | 운영 비용 증가 | 캐싱 + rate limit + 일일 한도 설정 |
| 기존 프로젝트 모듈 회귀 | 데이터 손실 | 마이그레이션 스크립트 + 롤백 계획 |
