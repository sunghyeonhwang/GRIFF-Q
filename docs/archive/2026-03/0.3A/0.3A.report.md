### 2.1 A1. 프로젝트 리워크 (34h)

#### 구현 사항
- ✅ 프로젝트 목록 3종 뷰 (List, Board, Calendar)
  - List: 테이블 형식 (이름, 상태, 진행률, 멤버, 마감일)
  - Board: 상태별 컬럼 (active/completed/on_hold)

- ✅ 프로젝트 생성 리워크 (4단계 Dialog)
  - Step 1: 종류 선택 (general/event/content/maintenance)
  - Step 2: 기본 정보 (이름, 설명, 시작/종료일)
  - Step 3: R&R 자동 로드 + 멤버 배정
  - Step 4: 확인 및 저장
- ✅ 프로젝트 상세 허브 탭 구성
  - Overview, Tasks, Dependency Map, Timeline, Settings, AI Review
- ✅ 진행률 자동 계산
  - DB Function: `calculate_project_progress()`
  - Trigger: Task 상태 변경 시 자동 업데이트
  - 계산식: 완료 Task 가중치 / 전체 Task 가중치 * 100
- ✅ 임무카드 자동 발급
  - 프로젝트 종류별 Task 템플릿 (4종류 × 4~6개 Task)

## 3. 기술 스택

- **UI Library**: shadcn/ui + Radix UI

- **행 보안**: PostgreSQL RLS

### 3.3 AI/LLM
- **AI 모델**: Gemini 2.0 Flash
- **통신**: SSE (Server-Sent Events)
- **프레임워크**: @google/generative-ai

## 7. Lessons Learned

### 7.1 What Went Well

1. **DB-First Design의 효율성**
   - 6개 마이그레이션을 처음부터 정교하게 설계하여 후속 변경 최소화
   - RLS 정책, Enum, Trigger를 사전에 정의하여 구현 중 혼선 없음

2. **dnd-kit 선택의 정확성**
   - React 19 호환성 완벽, 번들 크기 작음 (6KB)
   - 드래그 앤 드롭 UX가 부드럽고 안정적

3. **React Flow 의존성 맵의 우수성**
   - DAG 시각화에 최적, 커스텀 노드/엣지 지원 우수
   - dagre 통합으로 자동 정렬 구현 용이

4. **Gemini 2.0 Flash의 성능**
   - 응답 속도 매우 빠름 (0.5~2초)
   - 스트리밍 지원으로 UX 향상
   - 비용 대비 품질 우수

5. **설계 문서의 정확성**
   - Design 문서를 99% 이상 따라서 구현 가능
   - 의도적 변경 4건도 모두 UX 개선으로 정당화됨

6. **Gap Analysis의 2-Pass 전략**
   - v1.0 → v2.0 재분석으로 누락 항목 8/9 해결
   - 최종 Match Rate 97.6%에 도달

### 7.2 Areas for Improvement

1. **프로젝트 뷰 아키텍처**
   - Design에서 6종 뷰를 프로젝트 목록에 배치한 의도를 초기에 재검토 필요
   - v2.0에서 3종 (목록) + 5종 (Task 상세)로 최적화했으나, Design 문서 업데이트 필요

2. **Task 로딩 스켈레톤**
   - 기술 부채 1건 발생 (낮은 영향도)
   - 구현 체크리스트에 포함시켜 build 전에 완료 필요

3. **AI API 키 관리**
   - GEMINI_API_KEY 미설정 시 graceful fallback을 명시적으로 Design 문서에 기록 필요
   - 프로덕션 환경에서 API 키 누락 시나리오에 대한 테스트 강화

4. **성능 최적화 기회**
   - React Flow 캔버스: 노드 50개 이상 시 렌더링 성능 저하 가능
   - 향후 가상화(virtualization) 고려 필요

### 7.3 To Apply Next Time

1. **PDCA 문서 버전 관리**
   - v1.0 → v2.0 재분석 시 명확한 버전 표시 (이번처럼)
   - 각 단계 종료 시 별도 섹션 "Version History" 작성

2. **Design 문서 동적 갱신**
   - 구현 중 의도적 변경 발생 시 즉시 Design 문서에 마크 (`🔵 Changed`, `🟡 Added`)
   - 최종 리뷰 때 일괄 업데이트

3. **컴포넌트 checklist 강화**
   - 각 컴포넌트별 로딩/에러/스켈레톤 상태 구현 강제
   - `tasks/loading.tsx` 같은 누락 방지

4. **AI 통합 가이드**
   - Gemini 외 다른 LLM 사용 시 fallback 패턴 표준화
   - Rate Limit, Cost 분석을 Design 단계에 포함

5. **의존성 맵 성능**
   - React Flow 캔버스 최적화 (가상화, 레이아웃 캐싱)
   - 노드 50개 이상 프로젝트에 대한 성능 벤치마크 수행
