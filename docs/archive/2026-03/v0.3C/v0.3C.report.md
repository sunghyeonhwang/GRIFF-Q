### Design

**Design 문서**: `docs/02-design/features/v0.3C.design.md`

**주요 설계 결정**:

1. **아키텍처**:
   - Server Component (`/scrum/page.tsx`) → Client Component (`ScrumSession`) 분리
   - 병렬 데이터 페칭 (스크럼 + 이월 + 일정 + 마감 Task + 사용자 정보)
   - SSE 스트리밍으로 AI 대화 실시간 응답

2. **DB 설계**:
   - `daily_scrums`: 사용자별 일자 고유 스크럼 기록
   - `scrum_items`: 스크럼 항목 (우선순위, 시간블록, 출처 추적)
   - RLS 정책: 본인 + 관리자 역할 접근

3. **컴포넌트 계층**:
   - `ScrumSession`: 전체 상태 + 단계 진행 관리
   - `ScrumStepper`: 3단계 시각 인디케이터
   - `ScrumBrainstorm`, `ScrumPrioritize`, `ScrumSchedule`: 각 단계 UI
   - `ScrumAiChat`: Gemini SSE 통합
   - `ScrumTimeline`: 09:00~18:00 타임블록 시각화
   - `ScrumEisenhower`: 2×2 매트릭스 뷰
   - `ScrumSummary`, `ScrumReview`: 완료 및 회고

4. **AI 프롬프트**:
   - 할 일 수집 도우미 역할
   - 어제 미완료 항목 + 오늘 마감 Task + 오늘 일정 컨텍스트 포함
   - 자연어 대화 + 구조화된 목록 추출

5. **모바일 반응형**:
   - 데스크톱: 3단계 가로 표시 + 2×2 매트릭스 그리드
   - 모바일: 숫자만 표시 + 2×1 스택 레이아웃 + 터치 드래그 센서

## Lessons Learned

### What Went Well

1. **AI 컨텍스트 통합 성공**: Gemini에 어제 미완료, 오늘 마감, 오늘 일정을 함께 전달하여 AI 제안의 정확도 높음. 사용자 대화만으로 할 일 자동 추출 가능.

2. **3단계 플로우 직관성**: 브레인스토밍 → 우선순위 → 시간 배치의 순차적 구조가 사용자 경험을 자연스럽게 함. 각 단계 사이의 진행 상태도 명확함.

3. **dnd-kit 드래그 재사용**: v0.3A 킥오프 체크리스트에서 구축한 드래그 패턴을 그대로 재사용하여 구현 시간 단축. PointerSensor로 모바일 터치도 완벽 지원.

4. **타임라인 시각화**: 09:00~18:00을 30분 단위로 타일링하고, 스케줄 고정 블록 + 할당 블록을 색상으로 구분한 결과 시각적 일정 관리 매우 직관적.

5. **RLS 정책 명확화**: daily_scrums는 본인/관리자, scrum_items는 scrum 소유자만 접근 가능하도록 설정하여 보안 + 성능 동시 확보.

6. **SSE 스트리밍 안정성**: Gemini의 SSE 스트리밍을 proper error handling과 함께 구현하여 네트워크 오류 시에도 graceful하게 처리.

7. **검증 프로세스 효율성**: 96.3% → 100%로의 4개 갭 수정이 1일 내 완료되어 iteration 프로세스가 효과적임을 입증.

---

### Areas for Improvement

1. **AI 응답 파싱 정확도**: Gemini의 응답이 항상 구조화된 목록으로 나오지 않을 수 있음. 향후 JSON 형식 강제 또는 few-shot prompting으로 개선 필요.

2. **타임블록 드래그**: 현재 시간 슬롯을 팝오버로 선택하지만, 타임라인에서 직접 드래그로 배치하는 UX가 더 자연스러울 수 있음 (v0.3D 후속 개선 대상).

3. **다국어 지원**: 현재 한국어만 지원. 국제화 필요 시 i18n 통합 추가.

4. **성능 최적화**: 스크럼 이력이 30일 × 4 파일(daily_scrums + scrum_items) = 많은 데이터. 페이지 로드 시 무한 스크롤 또는 페이지네이션 추가 검토.

5. **마감 임박 Task 정의**: 현재 D-2를 기준으로 하나, PM 정책에 따라 조정 필요 (설정화 가능).

6. **알림 연동**: 현재 스크럼 미진행 시 PM에게 알림이 없음. v0.3D 인앱 알림 시스템 완료 후 통합.

1. **설계 검증 체크리스트**: 설계 단계에 "파일 목록 정확성", "파라미터 완전성", "에러 메시지 포맷" 등 세부 항목 추가. 초기 매치율 100% 달성 가능.

2. **AI 프롬프트 엔지니어링**: 컨텍스트 (이월 + 마감 + 일정) 구성 시 명확한 구분선을 두어 Gemini 이해도 향상 (이번 성공 사례 재확인).

3. **시간 배치 UX 선택**: 팝오버 vs 드래그 방식의 장단점을 초기 사용자 테스트로 확인 후 결정. 현재는 팝오버가 정확하나 드래그가 더 직관적일 가능성.

4. **이월 알고리즘**: "어제 미완료" 기준을 status='planned'가 아닌 'completed' 이외로 확대하여 더 포괄적으로. 현재는 동작하나 명확한 의도 주석 추가.

5. **Task 연동 추적**: scrum_item이 source_task_id 또는 generated_task_id를 가지도록 설계한 것이 양방향 추적을 가능하게 함. 이 패턴을 향후 Task 관계 설계에 적용.

6. **모바일 우선 테스트**: 375px 기준 모바일 QA를 초기부터 포함하면 반응형 버그 감소. 이번 4조합 QA를 정례화.
