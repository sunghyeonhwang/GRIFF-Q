### 2.4 진행률 자동 계산 (DB Function)

```sql
-- 015_tasks.sql (하단)
CREATE OR REPLACE FUNCTION calculate_project_progress(p_project_id uuid)
RETURNS float AS $$
DECLARE
  total_weight integer;
  completed_weight integer;
BEGIN
  SELECT
    COALESCE(SUM(weight), 0),
    COALESCE(SUM(CASE WHEN status = 'completed' THEN weight ELSE 0 END), 0)
  INTO total_weight, completed_weight
  FROM tasks
  WHERE project_id = p_project_id
    AND parent_task_id IS NULL;

  IF total_weight = 0 THEN
    RETURN 0;
  END IF;

  RETURN ROUND((completed_weight::float / total_weight * 100)::numeric, 1);
END;
$$ LANGUAGE plpgsql;

-- Task 상태 변경 시 프로젝트 진행률 자동 업데이트
CREATE OR REPLACE FUNCTION update_project_progress()
RETURNS trigger AS $$
BEGIN
  IF NEW.project_id IS NOT NULL THEN
    UPDATE projects
    SET progress = calculate_project_progress(NEW.project_id),
        updated_at = now()
    WHERE id = NEW.project_id;
  END IF;

  IF OLD.project_id IS NOT NULL AND OLD.project_id != COALESCE(NEW.project_id, '00000000-0000-0000-0000-000000000000') THEN
    UPDATE projects
    SET progress = calculate_project_progress(OLD.project_id),
        updated_at = now()
    WHERE id = OLD.project_id;
  END IF;

  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_update_project_progress
  AFTER INSERT OR UPDATE OF status, weight, project_id OR DELETE ON tasks
  FOR EACH ROW
  EXECUTE FUNCTION update_project_progress();
```

## 7. Data Flow

### 7.1 프로젝트 생성 플로우

```
[사용자] 프로젝트 생성 버튼 클릭
    ↓
[ProjectCreateDialog] 4단계 스텝 폼
    ↓
[createProjectWithTemplate()] Server Action
    ├── 1. projects INSERT
    ├── 2. project_members BULK INSERT (R&R 템플릿)
    ├── 3. tasks BULK INSERT (임무카드 템플릿)
    └── 4. task_dependencies INSERT (기본 의존성)
    ↓
[응답] { project, tasks, members }
    ↓
[Router] /projects/[id] 상세 페이지로 이동
```

### 7.2 Task 상태 변경 → 진행률 업데이트 플로우

```
[TaskCard] 드래그 or 상태 변경
    ↓
[reorderTasks() / updateTaskStatus()] Server Action
    ↓
[DB Trigger] update_project_progress()
    ↓
[projects.progress] 자동 갱신
    ↓
[revalidatePath] UI 반영
```

### 7.3 의존성 추가 플로우

```
[DependencyMapCanvas] 노드 간 드래그 연결
    ↓
[validateDependency()] 순환 참조 검증 (DFS)
    ├── 순환 있음 → 에러 토스트 + 연결 취소
    └── 순환 없음 ↓
[addDependency()] Server Action
    ↓
[Edge 추가] 캔버스 갱신
```

### 7.4 AI 대화 플로우

```
[ReviewChat] 사용자 메시지 입력
    ↓
[checkRateLimit()] 분당 10회 확인
    ├── 초과 → 안내 메시지
    └── 통과 ↓
[POST /api/projects/[id]/review/chat]
    ├── 1. 프로젝트 컨텍스트 빌드
    ├── 2. 대화 히스토리 로드
    ├── 3. Gemini 2.0 Flash 스트리밍 호출
    └── 4. 대화 저장
    ↓
[SSE Stream] 실시간 메시지 표시
```

## 10. Key Design Decisions

| 결정 | 선택 | 이유 |
|------|------|------|
| Task 상태 관리 | Server Action + DB Trigger | 진행률 자동 계산을 DB 레벨에서 보장 |
| Kanban 드래그 | dnd-kit | React 19 호환, 번들 크기 작음, 접근성 우수 |
| 의존성 맵 | @xyflow/react (React Flow) | DAG 시각화 최적, 커스텀 노드/엣지 지원 |
| DAG 레이아웃 | dagre | 방향 그래프 자동 배치 표준 라이브러리 |
| AI 모델 | Gemini 2.0 Flash | 빠른 응답 + 낮은 비용, 스트리밍 지원 |
| AI 통신 | SSE (Server-Sent Events) | 단방향 스트리밍에 최적, 웹소켓 대비 간단 |
| 진행률 계산 | DB Function + Trigger | 클라이언트 의존 없이 데이터 정합성 보장 |
| R&R 템플릿 | 프로젝트 종류별 기본값 | 빠른 프로젝트 시작, 수정 가능 |

## 11. Error Handling & Edge Cases

| 상황 | 처리 |
|------|------|
| DAG 순환 참조 | DFS 검증 → 에러 토스트 + 연결 거부 |
| Task 50개+ 노드 | 미니맵 표시 + 자동 정렬 권유 |
| 모바일 의존성 맵 | 터치 줌/패닝만 허용, 드래그 연결은 데스크톱 전용 |
| AI API 장애 | "AI 서비스 일시 중단" 안내 + 수동 보고서 작성 링크 |
| Rate Limit 초과 | 남은 시간 표시 + 재시도 안내 |
| 프로젝트 종류 변경 | 기존 Task 유지, 새 템플릿은 "추가" 옵션 |
| 미배정 역할 | "미배정" 표시 + 킥오프 경고 알림 |
| 선행 Task 미완료에서 진행 | 경고 토스트만 표시, 진행 허용 |
| Task 삭제 시 의존성 | ON DELETE CASCADE → Edge 자동 삭제 |
