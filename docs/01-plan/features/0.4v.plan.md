| **Problem** | 현재 대시보드는 금융(입금/견적/회고) 중심이며, v0.3A에서 추가된 업무 데이터(tasks, projects)가 반영되지 않아 업무 흐름 파악이 불가능하다. 또한 개인정보 수정 기능이 없어 프로필/비밀번호 변경 시 관리자 개입이 필요하다. |

| **Core Value** | 업무 중심 대시보드로 일일 업무 시작 시간 단축, 자기 관리형 프로필로 운영 효율 향상 |

## 1. Background

v0.3A에서 프로젝트/TASK/의존성맵/AI리뷰가 추가되었으나, 대시보드는 여전히 v0.25의 금융 중심 구성이다.
업무 데이터(tasks, projects, project_members)를 활용한 업무 중심 대시보드가 필요하다.

또한 현재 `/settings` 는 boss 역할의 사용자 관리 리다이렉트만 있고, 개인정보(이름/비밀번호) 수정 기능이 없다.

---

## 2. Scope

### 2.1 대시보드 개편

**탭 구조**: 업무 탭 (기본) / 금융 탭

#### 업무 탭 — 위젯 6종

| # | 위젯 | 설명 | 데이터 소스 | 비고 |
|---|------|------|-------------|------|
| 1 | **오늘의 흐름** | 오늘 마감 Task, 완료 Task, 전체 Task, 진행중 | tasks | v0.3A 데이터 활용 |
| 2 | **업무 달성도** | 개인 Task 완료율 (이번 주/이번 달) | tasks | 진행률 바 + 수치 |
| 3 | **리마인더** | 읽지 않은 알림 목록 (최신 5건) | notifications | 기존 테이블 활용 |
| 4 | **내 업무** | 나에게 할당된 Task Top 5 (우선순위 정렬) | tasks | 상태 뱃지 표시 |
| 5 | **프로젝트 진행률** | 활성 프로젝트별 진행률 바 | projects, tasks | DB 트리거 활용 |
| 6 | **워크로드** | 팀원별 할당 Task 수 분포 | tasks, users | boss/manager만 표시 |

> **제외 항목**: "데일리 스크럼" 위젯과 "오늘 미팅" 위젯은 daily_scrums/schedules 테이블이 v0.3B/C/D에서 구현 예정이므로 v0.4 범위에서 제외. v0.5 이후에 추가.

#### 금융 탭 — 기존 위젯 유지

| 위젯 | 현재 상태 |
|------|-----------|
| 인사말 + 날짜 | 유지 (탭 공통 헤더로 이동) |
| 퀵 액션 | 유지 (탭 공통) |
| 요약 카드 4종 | 유지 (마감 임박, 입금 대기, 최근 회의, 프로젝트) |
| 입금 추이 차트 | 유지 |
| 견적 상태 차트 | 유지 |
| 회고 제출 차트 | 유지 |
| 마감 임박 테이블 | 유지 |
| 입금 대기 테이블 | 유지 |
| 활동 타임라인 | 유지 |
| 최근 회의록 | 유지 |
| 회고 제출 현황 | 유지 |

### 2.2 프로필 세팅

| 기능 | 설명 | 비고 |
|------|------|------|
| 이름 수정 | users.name 변경 | Server Action |
| 비밀번호 변경 | 현재 비밀번호 확인 → 새 비밀번호 설정 | Supabase Auth |
| 프로필 이미지 | 아바타 업로드 (Supabase Storage) | 선택적 구현 |
| 알림 설정 | 알림 타입별 on/off | notifications 테이블 확장 |

## 3. 기술 설계 요약

### 3.1 DB 변경

**기존 활용**:
- `notifications` 테이블 (008_notifications.sql) — 이미 존재
- `tasks` 테이블 (015_tasks.sql) — v0.3A 구현 완료
- `projects` 테이블 (009, 012) — progress 필드 포함

**신규 마이그레이션**:
- `users` 테이블에 `avatar_url text` 추가 (이미 010에서 추가 여부 확인 필요)
- `user_settings` 테이블 신규 (알림 설정 저장)

```sql
-- user_settings 테이블
CREATE TABLE user_settings (
  user_id uuid PRIMARY KEY REFERENCES auth.users(id),
  notification_email boolean DEFAULT true,
  notification_push boolean DEFAULT true,
  notification_types jsonb DEFAULT '{}',
  updated_at timestamptz DEFAULT now()
);
```
