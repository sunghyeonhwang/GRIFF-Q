| **Problem** | 프로젝트 시작 시 팀원/목표/체크리스트를 체계적으로 설정하는 절차가 없어 킥오프가 구두로만 진행되고, 회사 일정(방학/생일/공휴일/회의)을 관리하는 캘린더가 부재하다. |

| **Core Value** | 프로젝트 시작 품질 향상(킥오프 프로세스 표준화), 회사 일정 가시성 확보(캘린더 통합 관리) |

## 1. Background

v0.3A에서 프로젝트/TASK/의존성맵/AI리뷰가 구현되었으나, 프로젝트 시작 절차(킥오프)와 회사 일정 관리는 여전히 부재하다.

현재 상태:
- **킥오프**: 프로젝트 생성 후 바로 태스크 생성 가능 — 목표/범위/팀 구성 없이 무계획 시작 위험
- **일정**: 회의록 모듈의 meeting_date만 존재 — 방학/생일/공휴일/회사 행사 등 전사 일정 관리 불가

## 2. Scope

### 2.1 B1. 프로젝트 킥오프

#### 핵심 기능

| 기능 | 설명 |
|------|------|
| 킥오프 페이지 | `/projects/[id]/kickoff` — 프로젝트 상세 탭으로 접근 |
| 프로젝트 개요 | 목표, 작업 범위, 제약 조건, 성공 기준 입력 |
| 팀 구성 | project_members R&R 매핑 (기존 데이터 연동) |
| 마일스톤 | project_milestones 타임라인 (기존 데이터 연동) |
| 체크리스트 | 시작 전 확인 항목 (담당자별 체크 + 자유 추가 + 드래그 정렬) |
| 미팅 기록 | 킥오프 미팅 참석자, 결정사항, 노트 |
| 완료 조건 | 모든 체크리스트 완료 시 "킥오프 완료" 버튼 활성화 |
| 태스크 생성 전제조건 | **"프로젝트" 유형**에서만 킥오프 완료 후 태스크 생성 가능 |

> **적용 범위**: "프로젝트" 유형에서만 킥오프 필수. 미니프로젝트/셀프프로젝트는 킥오프 없음.

#### 기본 체크리스트 (프로젝트 유형 공통, 자동 생성)

1. 프로젝트 목표 확정
2. 팀원 배정 완료
3. 마일스톤 설정 완료
4. 레퍼런스/자료 수집
5. 위험 요소 식별

#### 킥오프 플로우

```
프로젝트 생성 (유형: "프로젝트")
  → 킥오프 탭 활성화 (status: draft)
  → 개요/팀/마일스톤/체크리스트/미팅 기록 작성
  → 킥오프 완료 (status: completed)
  → 태스크 탭 활성화 (태스크 생성 가능)

※ 킥오프 미완료 시 태스크 생성 시도 →
  "킥오프를 먼저 완료해주세요" 안내 + 킥오프 탭 이동 유도
```

### 2.2 B2. 일정 캘린더

#### 핵심 기능

## 3. DB 스키마

### 3.1 project_kickoffs 테이블 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `project_id` | uuid | FK → projects (UNIQUE) |
| `objective` | text | 프로젝트 목표 |
| `scope` | text | 작업 범위 |
| `constraints` | text | 제약 조건 |
| `success_criteria` | text | 성공 기준 |
| `kickoff_date` | date | 킥오프 일자 |
| `meeting_notes` | text | 킥오프 미팅 기록 (리치 텍스트) |
| `meeting_attendees` | uuid[] | 미팅 참석자 ID 배열 |
| `meeting_decisions` | jsonb | 주요 결정사항 `[{title, content}]` |
| `status` | text | `draft` / `in_progress` / `completed` |
| `completed_at` | timestamptz | 킥오프 완료 시점 |
| `created_at` | timestamptz | 생성 시점 |
| `updated_at` | timestamptz | 수정 시점 |

### 3.2 kickoff_checklist_items 테이블 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `kickoff_id` | uuid | FK → project_kickoffs |
| `title` | text | 체크리스트 항목명 |
| `description` | text | 상세 설명 |
| `assignee_id` | uuid | FK → users |
| `is_completed` | boolean | 완료 여부 |
| `due_date` | date | 기한 |
| `sort_order` | integer | 정렬 순서 |

### 3.3 schedules 테이블 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid | PK |
| `title` | text | 일정명 |
| `description` | text | 설명 |
| `category` | text | `vacation` / `salary_review` / `birthday` / `holiday` / `company` / `meeting` / `other` |
| `start_date` | date | 시작일 |
| `end_date` | date | 종료일 (nullable: 당일 일정) |
| `is_all_day` | boolean | 종일 여부 (default: true) |
| `start_time` | time | 시작 시간 (is_all_day=false일 때) |
| `end_time` | time | 종료 시간 |
| `color` | text | 캘린더 표시 색상 |
| `created_by` | uuid | FK → users |
| `is_public` | boolean | 전사 공개 여부 (default: true) |
| `target_user_id` | uuid | FK → users (nullable: 개인 일정일 때) |
| `created_at` | timestamptz | 생성 시점 |

## 10. 리스크

| 리스크 | 대응 |
|--------|------|
| FullCalendar 번들 크기 | dynamic import + tree-shaking, 필요 플러그인만 설치 |
| FullCalendar 다크모드 CSS | Tailwind CSS 변수 기반 커스텀 테마 적용 |
| 킥오프 미완료 시 태스크 차단이 UX 혼란 | 명확한 안내 배너 + 킥오프 탭 바로가기 링크 |
| 미니프로젝트→프로젝트 업그레이드 시 킥오프 처리 | 업그레이드 시 킥오프 자동 생성, 기존 태스크는 유지 |
| 개인 일정 vs 전사 일정 혼동 | is_public 필드로 구분, 사이드바에 "내 일정만 보기" 토글 |
